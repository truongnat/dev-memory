# Guards and authentication

## ApiKeyGuard (user routes)

- Header: `x-api-key`
- Value format: `kb_live_<32-hex>` (generated by admin)
- Stored bcrypt-hashed in Neo4j; raw shown only at creation

```typescript
@Controller('kb')
@UseGuards(ApiKeyGuard)
export class KbController {}
```

Guard reads header, calls `AuthService.validateKey(key)`, throws `UnauthorizedException` if missing or invalid.

## MasterGuard (admin routes)

- Header: `x-master-password`
- Compared to env `MASTER_PASSWORD` (not stored in graph)

Use for key generation, destructive admin operations.

## Throttling

`ThrottlerGuard` as global `APP_GUARD`; override per route:

```typescript
@Throttle({ short: { limit: 2, ttl: 1000 } })
@Get('search')
search() {}
```

## Client example (CLI)

```typescript
headers: {
  'Content-Type': 'application/json',
  'x-api-key': config.api_key,
}
```

## Security rules

- Never log full API keys
- Never return hash or internal Neo4j errors to clients
- Use `UnauthorizedException` / `NotFoundException` — not generic 500 for auth failures
